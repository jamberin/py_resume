Personal Projects

Below list the different personal projects that I've worked on personally outside of the workplace. A brief description may be all I have of the project (or of what I'm willing to share at the moment) or it may include links to the github project or other external links.

Personal Website - Java and Python Versions

I first built a version of this site that was using a tomcat back end with a super basic contact form. I ended up needing to abandon the first version of that since I didn't have the time to manage it. I ended up reworking it, and got a stable version redeployed. Since then, I've learned a lot more about python, and I've been using that for this version of the site.

Catan - Python Application

I started this project because I was trying to play Catan on my android phone and it would constantly crash whenever I would launch the application. I figured I could just start building something that I can adapt to a phone application at some point. So far I've built the core engine that is a light python application that just works with a series of JSON elements. I'm now working on getting that integrated with an HTML element for a simple beta to test all the elements of that on, then I plan on building it into a phone application. I don't really intend on making this widely available, just something I want to build out slowly over time.

Configman - Flask Application

This project was started after I realized I wanted to have a central place to keep all of my properties for applications to consume, while also handling a custom tokenization system. I decided to go with a custom applicaiton instead of an OOB solution mostly for the challenge. If there is any need to build this out for futher uses, it's available for that kind of growth.

Employment Projects

I have worked on a number of projects for the companies I have worked for. Below will be a listing of some of the projects that I've been a part of and what role I had in the project. I will have to keep it discreet since I don't like being sued.

Middle Layer Application - Data Transformation

I was the lead QA on this application which took XML data from an external vendor's service and converted it into data for our internal Hybris system to consume. This had increased complexities when it came to testing due to the use of an external vendor's service, the use of existing systems for the transformation layer of the application, and the testing of both a .NET change and a Hybris change. We managed to get this application in place which enabled a lot of the users to hit the external vendor's service with little to no changes to their ordering work flow.

Locust Performance Testing - Python Framework

I worked alone on building out locust performance testing. I had to build out the framework similar to what already existed at the company in order to test the payloads because the PyTest framework the API library was built on didn't like to play well with others. I first built out the simple GET requests, worked into the static(ish) payloads for POSTs, and set up simple setups for Approve/Reject endpoints. This is something that will be built out further in the future.

Python Upgrade

Fortunately, there hasn't been too many issues yet, but each month it's getting progressively worse. There are certain layers of complexity, but I'm personally updating over 7 repositories and confirming the funcationality is working appropriately.
